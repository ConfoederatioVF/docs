Naissance versions are named after bodies of water. It is always recommended to use the latest version of Naissance, and to update as soon as possible to keep up with new capabilities and bugfixes.

 

Naissance ST vs. Naissance MT

Warning

Disclaimer: In-development information. These sections can fall out of date rapidly if not updated. Last updated: 30 June 2026.

Naissance ST (short for single-threaded) is the main operating version for Naissance GIS, whilst Naissance MT (short for multi-threaded) is the experimental branch which is currently a work-in-progress and used only for viewing .naissance files. Naissance MT's backend is largely complete, and only requires a frontend UI port.

Separate version names are not currently given for Naissance MT.

Vercengen

Vercengen is the main software engine utilised by Naissance, and is codenamed after technological/cultural eras. Although it is used for the main backend, frontend, and for modding and scripting, non-power users typically do not need to pay attention to its version.

If you are a power user, consider Installing the Dev Build instead, alongside Collation.
